## 1. 신규 추가 기능 개요

기존 발표자료 이후 IMI PRO에 추가된 주요 기능은 다음과 같습니다.

| 분류 | 신규 기능 | 비고 |
|------|-----------|------|
| 봇 제어 | **Firebase 원격 제어 채널** | 확장프로그램 없는 PC에서도 봇 즉시 제어 |
| 봇 제어 | **봇 연결 끊김 자동 감지** | 5분 이상 미응답 시 경고 표시 |
| 봇 제어 | **시작 시 즉시 상태 동기화** | 페이지 로드 즉시 확장프로그램 상태 반영 |
| 권한 | **부관리자 가입 승인 기능** | 기존 관리자 전용 → 부관리자도 처리 가능 |
| 권한 | **멤버 목록 전체 공개 + 탭별 권한 분리** | 역할에 따라 접근 가능한 탭 차별화 |
| 알림 | **감지 알림 긴급공지 자동 등록 버그 수정** | 모니터링 알림이 공지에 잘못 등록되는 문제 해결 |
| UI | **봇 중지 버튼 항상 활성화** | 이전에는 확장프로그램 연결 시에만 클릭 가능 |

## 2. IMI BOT 실시간 모니터링 시스템

IMI BOT 모니터링 시스템은 아이템매니아 거래 게시판에서 **특정 키워드·가격 조건에 맞는 판매글을 자동으로 탐지**하여, 실무자에게 즉각 알림을 전달하는 시스템입니다. 크롬 확장프로그램(물리 봇)과 웹 대시보드(IMI PRO)가 Firebase를 통해 실시간으로 연동됩니다.

---

### 2-1. 전체 아키텍처

```
┌─────────────────────────────────────────────────────────┐
│                   실무자 (웹 브라우저)                    │
│                                                         │
│   IMI PRO 웹 대시보드          크롬 확장프로그램          │
│   (GitHub Pages)         ↔    (IMI 물리 검색봇 v2.0)    │
│        │                             │                  │
│        │ bridge.js (메시지 브릿지)    │                  │
└────────┼─────────────────────────────┼──────────────────┘
         │                             │
         ▼                             ▼
┌─────────────────────────────────────────────────────────┐
│              Firebase Realtime Database                 │
│                                                         │
│  /bot_status     ← 봇 상태 실시간 업로드 (확장→웹)       │
│  /bot_cmd        ← 제어 명령 채널 (웹→확장)              │
│  /monitor_rules  ← 감시 규칙 공유 저장소                 │
│  /monitor_flash_state ← 감지 알림 트리거 (공유)          │
│  /monitor_history     ← 24시간 감지 이력 로그            │
│  /imi_blocked         ← 팀 공유 차단 목록               │
└─────────────────────────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────────────────────────┐
│              아이템매니아 웹사이트                        │
│         (itemmania.com — 감시 대상 페이지)               │
└─────────────────────────────────────────────────────────┘
```

**핵심 설계 원칙:**
- **중앙 Firebase DB**를 공유 상태 저장소로 활용 → 여러 PC·여러 브라우저에서 동일 상태 공유
- 확장프로그램이 없는 PC도 웹 대시보드만으로 봇 **원격 제어** 가능
- 한 실무자가 차단한 항목이 **팀 전체에 즉시 반영**

### 2-2. Firebase 제어 채널 (원격 봇 제어)

기존에는 크롬 확장프로그램이 설치된 PC에서만 봇을 제어할 수 있었습니다. 이번 업데이트에서 **Firebase `/bot_cmd` 경로를 제어 채널로 활용**하여, 확장프로그램이 없는 PC(또는 다른 브라우저)에서도 봇을 즉시 중지/시작할 수 있게 되었습니다.

#### 동작 흐름

```
웹 대시보드 [봇 중지] 클릭
       │
       ├─→ Firebase /bot_cmd 에 { cmd: 'stop', ts: 현재시각 } 기록
       │
       └─→ (로컬 확장프로그램이 연결된 경우) bridge.js 경유 즉시 전송
                      │
                      ▼
확장프로그램 background.js (Service Worker)
  - 1분마다 imi_rule_sync 알람 실행 시 /bot_cmd 조회
  - cmd.ts > lastHandled 이면 명령 실행
  - stopAll() / startSelected(ruleIds) 호출
  - 처리 후 /bot_status 갱신 → 대시보드에 즉시 반영
```

#### 핵심 코드 (`background.js`)

```javascript
async function checkBotCmd() {
    const cmd = await fireGet('/bot_cmd');
    if (!cmd || !cmd.ts) return;
    const lastHandled = await store.get('imi_cmd_ts').then(v => v || 0);
    if (cmd.ts <= lastHandled) return;   // 이미 처리한 명령은 무시
    await store.set('imi_cmd_ts', cmd.ts);
    if (cmd.cmd === 'stop') {
        await stopAll();
    } else if (cmd.cmd === 'start' && Array.isArray(cmd.ruleIds)) {
        await startSelected(cmd.ruleIds);
    }
}
```

- **중복 실행 방지:** `ts` 타임스탬프와 `imi_cmd_ts` 비교로 동일 명령 재실행 차단
- **최대 지연:** 알람 주기 1분 → 확장프로그램이 있는 PC는 bridge.js로 즉시, 없는 PC는 최대 1분 이내 반영

### 2-3. 봇 상태 대시보드 (실시간 동기화)

웹 대시보드의 모니터링 모달(`monitorModal`)은 Firebase `/bot_status`를 실시간 구독(`on('value', ...)`)하여 아래 정보를 자동 갱신합니다.

#### 표시 정보

| 항목 | 내용 |
|------|------|
| 상태 배지 | `● 감시 중` (녹색) / `■ 중지됨` (회색) / `⚠ 연결 끊김` (황색 경고) |
| 마지막 동기화 | 확장프로그램이 마지막으로 상태를 업로드한 시각 |
| 활성 규칙 수 | `감시 중 — N개 규칙 실행 중` |
| 규칙 목록 | 각 규칙별 키워드, 최소가격, 스캔 주기, 상태 표시 |

#### 연결 끊김 자동 감지

```javascript
var isStale = s.lastUpdate && (Date.now() - s.lastUpdate) > 5 * 60 * 1000;
// 5분 이상 갱신 없으면 "연결 끊김" 경고 표시
```

확장프로그램이 비정상 종료되거나 네트워크 단절 시, 마지막 업데이트로부터 **5분 경과** 시점에 자동으로 경고 상태로 전환됩니다.

#### 시작 시 즉시 동기화

```javascript
// bridge.js 연결 감지 즉시 → 확장프로그램에 상태 동기화 요청
if (!wasConnected) _sendToBot({ type: 'SYNC_STATUS' });
```

페이지를 새로 열거나 새로고침할 때 확장프로그램 연결이 감지되는 즉시 현재 봇 상태를 Firebase에 기록하도록 요청, 대시보드가 항상 최신 상태를 표시하도록 보장합니다.

### 2-4. 감시 규칙(Rule) 관리

감시 규칙은 Firebase `/monitor_rules` 경로에 저장되며, 관리자 비밀번호 인증 후 추가/삭제할 수 있습니다.

#### 규칙 구성 요소

| 필드 | 설명 | 예시 |
|------|------|------|
| `name` | 규칙 이름 | "리니지M 다이아 매입" |
| `url` | 감시할 아이템매니아 목록 URL | `https://www.itemmania.com/sell/...` |
| `keyword` | 감시 키워드 (콤마 구분, 다중 가능) | "다이아,다이아몬드" |
| `minPrice` | 최소 가격 조건 (원) | 500000 |
| `maxPages` | 스캔할 최대 페이지 수 | 3 |
| `gameLabel` | 게임명 태그 | "리니지M" |
| `enabled` | 활성 여부 | true / false |

#### 규칙 상태 표시

각 규칙은 아래 세 가지 상태로 표시됩니다:

- `● 감시중` (녹색): 규칙 활성화 + 감시 탭 열림
- `○ 대기` (황색): 규칙 활성화 + 탭 미오픈 (재시작 대기 중)
- `■ 비활성` (회색): 규칙 비활성화 상태

#### 테스트 기능

규칙 등록 전 실제 페이지에서 파싱 결과를 미리 확인할 수 있는 **규칙 테스트** 기능을 제공합니다.

```
[🔍 테스트] 버튼 클릭
       │
       ▼
최대 3페이지 순차 스캔
       │
       ▼
전체 파싱된 물품 수 / 조건 일치 물품 수 / 감지된 물품 목록 팝업 표시
```

### 2-5. 페이지 스캔 엔진

#### 물리 봇 스캔 (`bot.js` — 확장프로그램)

크롬 확장프로그램의 `bot.js`가 아이템매니아 실제 페이지의 DOM을 직접 파싱합니다.

**스캔 방식:**
- `li`, `tr`, `.item_row`, `.item_wrap` 요소를 순회
- 텍스트 블록에서 키워드 매칭 + 가격 추출
- 조건 일치 항목 감지 시 Firebase `/monitor_flash_state`에 알림 기록
- 스캔 간격: 규칙별 `scanInterval` 설정값 (기본 5초)

**가격 파싱 로직:**
```
쉼표 구분 숫자  → "1,500,000" → 1,500,000원
5자리 이상 숫자 → "150000" → 150,000원
만원 단위       → "150만원" → 1,500,000원
억원 단위       → "1억원"  → 100,000,000원
```

**탭 자동 재오픈:**
감시 탭이 실수로 닫히면 background.js가 2초 후 자동으로 새 탭을 열어 감시를 재개합니다.

#### 웹 스캔 (`monitor.js` — 대시보드)

웹 대시보드에서도 독립적으로 동작하는 스캔 엔진을 내장합니다. CORS 우회 프록시를 통해 아이템매니아 페이지를 직접 가져와 파싱합니다 (확장프로그램 없이도 동작).

### 2-6. CORS 우회 다중 프록시 시스템

아이템매니아는 외부 도메인에서의 직접 접근을 차단합니다. 이를 우회하기 위해 **계층적 폴백 방식**으로 총 4가지 접속 경로를 순차 시도합니다.

```
1순위: 탬퍼몽키(TamperMonkey) 우회 스크립트
       → 연결된 경우 가장 안정적 (세션 유지 가능)
       
2순위: 브라우저 직접 접속 (CORS 확장프로그램 활성화 시)
       → 8초 타임아웃, EUC-KR 인코딩 자동 처리
       
3순위: allorigins.win 무료 프록시
       → 응답에서 'contents' 필드 파싱

4순위: corsproxy.io → 실패 시 codetabs.com 순차 시도
       → 각 12초 타임아웃
       
모든 경로 실패 → 사용자에게 오류 메시지 표시
```

**EUC-KR 인코딩 처리:**
아이템매니아는 EUC-KR 인코딩을 사용하므로, 검색어를 EUC-KR로 인코딩하여 POST 요청해야 합니다. WHATWG 인코딩 스펙 테이블을 동적으로 로드하여 변환합니다.

### 2-7. 알림 시스템 (4중 알림)

감지 이벤트가 발생하면 **4가지 경로로 동시에 알림**을 전달합니다.

#### (1) 화면 플래시 오버레이

화면 전체가 붉은색으로 깜빡이는 시각적 경고 (0.25초 간격, 총 4회 반복)

```javascript
// rgba(239,68,68,0.35) ↔ rgba(239,68,68,0) 를 250ms 간격으로 8회 토글
window._overlayFlashInterval = setInterval(function() {
    count++;
    overlay.style.background = count % 2 === 1
        ? 'rgba(239,68,68,0.35)'
        : 'rgba(239,68,68,0)';
    if (count >= 8) clearInterval(window._overlayFlashInterval);
}, 250);
```

#### (2) 탭 제목 깜빡임

다른 탭에 있는 실무자도 감지할 수 있도록, 브라우저 탭 제목이 700ms 간격으로 전환됩니다.

```
"IMI PRO" ↔ "🚨 [3개 감지] 리니지M 다이아 매입"
```

#### (3) OS 브라우저 알림 (Notification API)

브라우저를 최소화하거나 다른 앱을 사용 중이어도 시스템 알림이 표시됩니다.

```javascript
new Notification('🚨 IMI PRO 사기글 감지', {
    body: '3개 물품 감지됨 — IMI PRO 확인',
    icon: 'https://msapi7890.github.io/IMI-PRO/favicon.ico'
});
```

#### (4) 확장프로그램 팝업 창

감지 발생 시 크롬 확장프로그램이 540×440px 팝업 창을 자동으로 띄웁니다. 5초 이내 중복 알림은 차단되어 스팸 방지.

#### 알림 패널 정보

- 규칙명, 게임명, 키워드, 최소가격 조건
- 감지된 물품 수
- 상위 3개 물품 목록 (제목 + 가격)
- 각 물품별 [필터제외] 버튼 (클릭 즉시 팀 공유 차단 목록에 등록)

---

### 2-8. 차단(블랙리스트) 필터

반복적으로 감지되는 정상 판매글 또는 불필요한 항목을 **팀 공유 차단 목록**에 등록하여 이후 알림에서 제외할 수 있습니다.

#### 차단 등록 경로
1. 감지 알림 패널의 [필터제외] 버튼
2. 감지 로그 패널의 [필터제외] 버튼
3. 크롬 확장프로그램 오버레이의 [🚫 차단] 버튼

#### 공유 메커니즘

- Firebase `/imi_blocked` 경로에 `{ key, title, addedBy, addedAt }` 형태로 저장
- 확장프로그램 background.js가 1분 주기로 Firebase 차단 목록 동기화
- **30일 경과 시 자동 초기화** (쌓이는 데이터 방지)

#### 차단 해제

웹 대시보드 로그 패널 > 차단목록 탭에서 개별 해제 또는 전체 해제 가능. 해제 즉시 Firebase 갱신 → 확장프로그램에 반영.

---

### 2-9. 감지 로그 패널

감지 이력을 **24시간** 동안 Firebase에 보관하며, 웹 대시보드의 로그 패널에서 조회할 수 있습니다.

#### 탭 구성

| 탭 | 내용 |
|----|------|
| 감지 로그 | 24시간 이내 모든 감지 이력 (시간대별 그룹화) |
| 차단 목록 | 현재 등록된 필터 목록 (등록자, 등록일시 표시) |

## 3. 크롬 확장프로그램 구조

IMI 물리 검색봇 v2.0은 Manifest V3 기반의 크롬 확장프로그램입니다.

### 파일 구성

| 파일 | 역할 |
|------|------|
| `manifest.json` | 확장프로그램 설정 (권한, 컨텐츠 스크립트 등록) |
| `background.js` | Service Worker — Firebase 연동, 탭 관리, 명령 처리 |
| `bot.js` | 아이템매니아 페이지 DOM 스캔 콘텐츠 스크립트 |
| `bridge.js` | IMI PRO ↔ background.js 메시지 중계 |
| `popup.html/js` | 확장프로그램 아이콘 클릭 시 표시되는 팝업 |
| `alert_popup.html/js` | 감지 시 자동으로 열리는 별도 알림 팝업 창 |
| `referrer_fix.js` | application.html 접근 시 Referer 헤더 자동 처리 |
| `rules.json` | declarativeNetRequest 규칙 (네트워크 요청 수정) |

### 메시지 통신 구조

```
IMI PRO 웹 대시보드
       │ window.postMessage({ __imiBot: true, type: 'STOP_ALL' })
       ▼
 bridge.js (content script)
       │ chrome.runtime.sendMessage({ type: 'STOP_ALL' })
       ▼
 background.js (Service Worker)
       │ stopAll() 실행 → 모든 감시 탭 종료
       │ syncStatus() → Firebase /bot_status 갱신
       ▼
 Firebase /bot_status
       │ 실시간 구독 (db.ref.on('value'))
       ▼
 IMI PRO 대시보드 UI 자동 갱신
```

### 탭 자동 관리

```
background.js의 탭 생존 확인 (3분 주기 알람)
       │
       ├ isActive() === true
       │       │
       │       └ startAll() → 닫힌 탭 자동 재오픈
       │
       └ isActive() === false → 아무 동작 없음

탭이 수동으로 닫힐 경우:
chrome.tabs.onRemoved → 2초 후 해당 탭 자동 재오픈 (감시 중일 때만)
```

## 4. 권한 체계 및 접근 제어

IMI PRO는 3단계 역할 기반 접근 제어를 구현합니다.

| 역할 | 봇 제어 | 규칙 추가/삭제 | 가입 승인 | 멤버 목록 | 긴급공지 |
|------|---------|---------------|----------|-----------|---------|
| **관리자(admin)** | ✅ | ✅ | ✅ | ✅ | ✅ |
| **부관리자(subadmin)** | ✅ | ✅ | ✅ | ✅ | ✅ |
| **일반 멤버** | ❌ | ❌ | ❌ | ✅ (열람만) | ❌ |

- 봇 제어 버튼은 **관리자/부관리자에게만 표시** (일반 멤버는 상태 조회만 가능)
- 규칙 추가/삭제 시 **관리자 비밀번호 입력** 추가 인증 요구

## 5. 기타 신규 기능 요약

### 멤버 관리 개선

- 멤버 목록 **전체 공개**: 역할과 관계없이 팀원 전체 목록 열람 가능
- **탭별 권한 분리**: 각 탭의 기능별로 접근 권한을 세밀하게 설정

### 부관리자 권한 확대

- 기존: 가입 승인은 관리자만 가능
- 변경: 부관리자도 신규 가입 요청 승인/거절 처리 가능

### 알림 버그 수정

- **문제**: 사기글 감지 알림이 긴급공지 게시판에 자동 등록되는 버그
- **원인**: `monitor_flash_state` 처리 로직이 `urgent_notices` 경로에 잘못 쓰기
- **해결**: 감지 알림은 `monitor_flash_state` 전용 경로로만 처리하도록 분리
